1 . As you can see, there is a recurring burst that appears in the time vs. particle count. These bursts occur approximately 5400 seconds apart, which is equal to one full 90 minute orbit. The plots above are of earth longitude, particle counts and solar phase vs time, respectively. With the middle graph, you can clearly see how the particle burst is related to the corresponding earth longitude and solar phase. These graphs were cut down to show about a 65th of the total data which allows us to see the patterns easily. The rest of the data follows this general pattern. This is signal contamination compared to the background.

Since the data follows a number of events per set interval, the distribution for the background particle counts is poisson. The resulting function would be a probability mass function, not density.

2 . The background changes as the satellite passes over a certain earth longitude. The background increases somewhat like a burst every 90 minutes, or once every 5400 seconds. This burst exponentially decays back down to the average particle count value in around half the cycle time of 5400 seconds.

3 . Using a burst/exponential decay function of: $${flux(time)} = A\exp(-\alpha t) $$ we can fill in the values to create a model for the background as a function of time. The only thing that changes with time is the time of the burst, which we can enter manually or via a loop.

def burst(x, b0, A1, T1, alpha1):
    """Burst model"""
    y = np.empty(x.shape)
    y.fill(b0)
    mask1 = (x >= T1)
    y[mask1] += A1 * np.exp(-alpha1 * (x[mask1] - T1))
   
    return y

zz = burst(time[0:82000]-9.4068e8,7,12,4150,.0003)

plt.plot(time[0:82000],pcounts[0:82000],linewidth=.5)
plt.plot(time[0:82000],zz);

Shown above is the fit model for a single burst, the following n bursts occur at about n*5400 seconds later.

2 . There doesn't seem to be much signal contamination apart from the few stars seen throughout the images. The background is not really time independent as it does not change as a function of time, there will always be that absence of light (at least for a while). It is pretty consistent spatially, so I'm going to go with a normal distribution for the pdf of the background. By averaging each image's pixel value, then averaging those 10 values I will get a good estimate of the average background value to use to create a normal distribution pdf.

for i in range(0,10):
    avg[i] = np.mean(imdata[0:199,0:199,i])

x = np.linspace(-4,4,201)
pdf = stats.norm.pdf(x,ov_avg)
plt.plot(x,pdf);

Subtracting the average from all the images should only leave large values left greater than zero for supernovae or other bright events to be visible. Whichever images have the largest values in any pixel clearly have an event occuring in them. By comparing that measurement value to a sigma, significance can be claimed or the measurement can be discarded.
